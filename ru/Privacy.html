Политика конфиденциальности EliteIntel

Данная политика объясняет, какие данные обрабатываются, как они используются и какие возможности выбора доступны пользователю.

Последнее обновление: 25 октября 2025 года

Обзор

EliteIntel приложение с открытым исходным кодом, доступное на GitHub. Оно использует преобразование речи в текст (STT), текста в речь (TTS) и большие языковые модели (LLM) для обработки игровых данных.

Приложение может работать полностью в автономном режиме с использованием локального STT (NVIDIA Parakeet), локального LLM (Ollama) и локального TTS (Kokoro). В этой конфигурации никакие данные не покидают устройство. При использовании облачных сервисов данные передаются, как описано ниже.

Какие данные обрабатываются?

Никакие личные данные не собираются: ни имена, ни адреса, ни геолокация. Обрабатываются следующие типы данных:

  • API-ключи: используются для аутентификации запросов к облачным TTS- и LLM-сервисам. Хранятся в зашифрованном виде в локальной базе данных SQLite. Передаются только в заголовках запросов к соответствующим сервисам (Google для TTS; xAI, OpenAI или Anthropic для LLM).

  • Текстовые данные (TTS): при использовании Google TTS текст ответа отправляется в Google. При использовании Kokoro TTS данные не покидают устройство.

  • Игровые данные (LLM): соответствующие игровые данные (детали миссий, рыночные данные, результаты сканирований и т. д.) отправляются в настроенный LLM. Имя Командира никогда не передаётся. ИИ обращается к вам по настроенному титулу, обращению или псевдониму.

Как используются эти данные?

  • API-ключи: хранятся в локальной базе данных и используются исключительно для аутентификации запросов к сторонним сервисам.

  • Аудио и текст: отправляются в Google только для обработки TTS. Google обрабатывает данные согласно своей собственной политике конфиденциальности. Хранение данных для улучшения сервиса по умолчанию отключено при стандартном использовании API.

  • Игровые данные: приложение не передаёт все игровые события в LLM. Оно собирает и хранит соответствующие данные локально, а затем отправляет целевые фрагменты при выдаче команды или запроса. LLM не имеет постоянного доступа к игровым данным.

Куда передаются данные?

  • Google (TTS): текст отправляется в облачные сервисы Google. Регулируется Политикой конфиденциальности Google.

  • xAI / OpenAI / Anthropic (LLM): фрагменты игровых данных отправляются в настроенный облачный LLM. Регулируется соответствующими политиками конфиденциальности.

  • Никуда больше: данные не хранятся за пределами устройства, не продаются и не передаются третьим лицам. Полный исходный код доступен на github.com/stone-alex/EliteIntel.

Права и выбор

  • Изучить код: полный исходный код доступен на GitHub.
  • Перейти в полностью автономный режим: использовать Parakeet, Ollama или LM Studio и Kokoro. В этой конфигурации данные не передаются.
  • Настроить провайдеров: выберите, какой облачный LLM использовать, если вообще использовать. API-ключи управляются в локальной базе данных.
  • Удалить данные: никакие данные не хранятся за пределами устройства. Для данных, хранящихся у Google, xAI, OpenAI или Anthropic, обратитесь к их соответствующим политикам.

Безопасность

API-ключи хранятся в зашифрованном виде в локальной базе данных и передаются только в заголовках запросов. Приложение соответствует Условиям обслуживания Elite Dangerous. Оно не читает игровую память и не использует оверлеи. Открытый исходный код позволяет сообществу проводить аудит.

Изменения политики

Обновления политики отмечаются в репозитории GitHub и могут появляться в приложении. Следите за изменениями на github.com/stone-alex/EliteIntel.
